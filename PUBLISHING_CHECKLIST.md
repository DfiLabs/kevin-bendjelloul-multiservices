## Publishing checklist (KB MultiServices)

### Contact
- [ ] Téléphone correct (tap-to-call OK sur mobile)
- [ ] WhatsApp OK (ouvre le bon chat + message pré-rempli)
- [ ] Email OK (le formulaire `mailto:` ouvre avec le bon `to:`)

### Services
- [ ] Les services listés sont bien ceux que vous voulez afficher (multi-services)
- [ ] Les exemples (3 bullets) correspondent à la réalité
- [ ] Les délais annoncés sont réalistes
- [ ] La partie “Nettoyage fin de chantier” est OK (ou la retirer)

### Légal / conformité
- [ ] Mentions légales correctes
- [ ] Confidentialité cohérente avec l’usage réel (pas de serveur)
- [ ] Cookies: cohérent (pas d’analytics = pas de bannière)
- [ ] Cohérence APE / positionnement (multi-services vs activité déclarée)

### UX / QA
- [ ] CTA OK: “Demander un devis”, “Appeler”, “WhatsApp”
- [ ] Mobile testé (iPhone + Android)
- [ ] Lisibilité OK (contraste, champs de formulaire)
- [ ] Scroll effects OK (pas trop lourd sur mobile)

### SEO / structured data
- [ ] JSON-LD type OK (HomeAndConstructionBusiness / LocalBusiness)
- [ ] Zone OK (Béziers + Hérault)
- [ ] FAQ cohérente avec la page
- [ ] Aucune adresse email affichée en clair
